# Poster. v3 — 行為流程圖（非技術版）

**給誰看**：合夥人、設計師、編輯者、投資人——任何沒寫程式的人。
**對應技術版**：`plan-v3-collection-pivot.md`（裡面有 SQL、API、schema）

---

## 一張圖講完整個產品

```
┌────────────────────────────────────────────────────────────────┐
│                                                                  │
│                    POSTER. — 海報收藏社群                         │
│                                                                  │
│                                                                  │
│   ┌──────────────┐       ┌──────────────┐       ┌──────────────┐ │
│   │              │       │              │       │              │ │
│   │  官方團隊    │ ────▶ │   資料庫     │ ◀──── │  一般使用者   │ │
│   │  (建資料)     │       │   (唯一真相) │       │  (玩 / 收藏)  │ │
│   │              │       │              │       │              │ │
│   └──────────────┘       └──────────────┘       └──────────────┘ │
│      建立目錄             海報 / 分類 / 收藏       翻牌 / 炫耀      │
│      上傳真實海報圖       狀態的同一套資料         補建官方沒有的   │
│                                                                  │
└────────────────────────────────────────────────────────────────┘
```

**三句話懶人包：**

1. **官方建立目錄**：非技術編輯者在 Google 試算表寫下海報資料，之後官方後台補上真實海報圖。
2. **使用者來玩**：看到滿坑滿谷的海報分類樹。他自己擁有的就**翻牌解鎖**（免審核、信任制），可以選擇附上自己拍的照片炫耀。
3. **雙邊協作**：使用者遇到官方還沒建檔的海報？可以在 app 內**提交建檔申請**（只填文字、不附圖），官方審核後補圖並加進資料庫。

## 第三段：隱私邊界（很重要）

非技術夥伴常會問：「那別人看得到什麼？」

```
┌─── 隱私邊界 ──────────────────────────────────────┐
│                                                      │
│                    自己看自己              別人看我   │
│                    ─────────              ─────────  │
│                                                      │
│  翻過的海報       ✓ 看得到              ✓ 看得到     │
│  (用官方圖)        (用我的自拍)           (用官方圖)   │
│                                                      │
│  翻過的海報       ✓ 看得到              ✓ 看得到     │
│  (未附自拍)        (用官方圖)            (用官方圖)   │
│                                                      │
│  沒翻過的海報     ✓ 看得到 (剪影)        ✗ 看不到    │
│                   (誘你去翻)              (不顯示)    │
│                                                      │
│  我拍的收藏照片   ✓ 看得到              ✗ 看不到    │
│                                          (永遠私密)   │
│                                                      │
│  收藏進度 / 稀有度 ✓ 看得到              ✓ 看得到     │
│  ── 例如「Henry 收藏了 42 張吉卜力海報」               │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**結論：**
- 「我有什麼」→ **公開**（炫耀感）
- 「我的收藏照長怎樣」→ **私密**（審美統一 + 版權安全）
- 「我還沒收到什麼」→ **隱藏**（不製造焦慮）

## 第四段：為什麼這樣設計？

### 為什麼不強制照片證明？

**因為收藏者會退坑。**

想像一個收藏了 500 張海報的資深玩家：

```
  強制證明版：
    → 登入 app → 要他一張張拍照 → 等審核 → 放棄
    → 3 天後卸載

  信任翻牌版：
    → 登入 app → 一次翻 50 張 → 秒解鎖成就 → 繼續
    → 他很爽，開始追更多缺的那些
```

**業界實證**：Letterboxd 不要你傳電影票根、Discogs 不要你拍唱片封套、MyAnimeList 不要你上傳截圖——**全部信任制**，全部數千萬使用者。這類 app 已經證明「信任 + 自願炫耀」比強制證明更能留住玩家。

### 為什麼把照片設為私密？

**三個理由：**

1. **審美統一** — 官方圖色調一致、品質整齊；如果每人都放自己拍的爛照片，profile 看起來像雜貨箱。
2. **版權安全** — 海報圖有版權、你拍的照片也可能不小心侵權（背景入鏡、場景重製等）。不公開就不牽涉法律。
3. **炫耀不用視覺** — 炫耀的是「我有 42 張吉卜力」這個事實，不需要把照片昭告天下。

### 為什麼別人看不到「你還沒收到的」？

**避免 FOMO 焦慮**。OpenSea 秀光所有 NFT，未擁有的人焦慮。Poster. 只秀你的成就，不秀你的缺口——別人看你 profile 看到的是「哇他好猛」，不是「他缺超多欸」。

## 一頁備忘（給合夥人會議用）

- **產品定位**：海報收藏者的卡夾 + 共建目錄社群
- **核心互動**：翻牌（信任制、免審核）
- **遊戲化**：進度條 + 稀有度 + 成就徽章 + 活動 feed
- **隱私**：我有什麼公開，我的自拍私密，我缺什麼隱藏
