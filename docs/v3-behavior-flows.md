# Poster. v3 — 行為流程圖（非技術版）

**給誰看**：合夥人、設計師、編輯者、投資人——任何沒寫程式的人。
**對應技術版**：`plan-v3-collection-pivot.md`（裡面有 SQL、API、schema）

---

## 一張圖講完整個產品

```
┌────────────────────────────────────────────────────────────────┐
│                                                                  │
│                    POSTER. — 海報收藏社群                         │
│                                                                  │
│                                                                  │
│   ┌──────────────┐       ┌──────────────┐       ┌──────────────┐ │
│   │              │       │              │       │              │ │
│   │  官方團隊    │ ────▶ │   資料庫     │ ◀──── │  一般使用者   │ │
│   │  (建資料)     │       │   (唯一真相) │       │  (玩 / 收藏)  │ │
│   │              │       │              │       │              │ │
│   └──────────────┘       └──────────────┘       └──────────────┘ │
│      建立目錄             海報 / 分類 / 收藏       翻牌 / 炫耀      │
│      上傳真實海報圖       狀態的同一套資料         補建官方沒有的   │
│                                                                  │
└────────────────────────────────────────────────────────────────┘
```

**三句話懶人包：**

1. **官方建立目錄**：非技術編輯者在 Google 試算表寫下海報資料，之後官方後台補上真實海報圖。
2. **使用者用它記錄自己的實體收藏**：看到滿坑滿谷的海報分類樹。他自己擁有的就**翻牌加入個人卡夾**（信任制、零審查、零徽章遊戲化）。想拍照做紀錄是 OK 的，但純私密、沒獎勵。
3. **雙邊協作**：使用者遇到官方還沒建檔的海報？可以在 app 內**提交建檔申請**（只填文字、不附圖），官方審核後補圖並加進資料庫。**貢獻建檔是唯一會產生徽章的行為**——因為它是唯一有自然閘門、無法造假的貢獻。

## 第三段：隱私邊界（很重要）

非技術夥伴常會問：「那別人看得到什麼？」

```
┌─── 隱私邊界 ──────────────────────────────────────┐
│                                                      │
│                    自己看自己              別人看我   │
│                    ─────────              ─────────  │
│                                                      │
│  翻過的海報       ✓ 看得到              ✓ 看得到     │
│  (用官方圖)        (用我的自拍)           (用官方圖)   │
│                                                      │
│  翻過的海報       ✓ 看得到              ✓ 看得到     │
│  (未附自拍)        (用官方圖)            (用官方圖)   │
│                                                      │
│  沒翻過的海報     ✓ 看得到 (剪影)        ✗ 看不到    │
│                   (誘你去翻)              (不顯示)    │
│                                                      │
│  我拍的收藏照片   ✓ 看得到              ✗ 看不到    │
│                                          (永遠私密)   │
│                                                      │
│  收藏進度 / 稀有度 ✓ 看得到              ✓ 看得到     │
│  ── 例如「Henry 收藏了 42 張吉卜力海報」               │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**結論：**
- 「我有什麼」→ **公開**（炫耀感）
- 「我的收藏照長怎樣」→ **私密**（審美統一 + 版權安全）
- 「我還沒收到什麼」→ **隱藏**（不製造焦慮）

## 第四段：為什麼這樣設計？

### 為什麼「翻牌」信任制？為什麼沒有徽章？

**因為物理海報根本無法驗證。**

如果我能驗證——徽章就有意義，那就發。但：

- 拍照：10 秒 Google 盜圖解決
- GPS：只能抓「未來」戲院發海報，歷史海報沒救
- NFC / QR：要 IP 方合作、不可能
- 區塊鏈：一樣要 IP 方發 token

**能免費又有效的驗證方法：不存在。** 這是物理海報（大量生產、無唯一識別、散佈無法追蹤）的根本屬性。

所以我們走 **Discogs / Letterboxd 路線**：信任制、工具導向、無徽章壓力。

想像一個收藏了 500 張海報的資深玩家：

```
  強制證明版：
    → 登入 app → 要他一張張拍照 → 等審核 → 放棄
    → 3 天後卸載

  信任翻牌版：
    → 登入 app → 一次翻 50 張 → 快速把藏品記錄完 → 繼續
    → 工具有用，留下來
```

**業界實證**：Discogs 800 萬使用者、Letterboxd 千萬使用者、MyAnimeList、TMDb——**全部信任制、沒有徽章競賽**。他們證明「好用的工具 + 完整的目錄 + 社群氛圍」不需要遊戲化裝飾。

### 為什麼沒有「翻了 100 張」、「集滿吉卜力」、「稀有度」這些徽章？

**因為它們全部可以秒破解。**

翻牌是免費的、無成本的點擊。任何建立在「翻牌數量」上的徽章，都等於**獎勵你快速點擊**——真收藏家不會來 farm 這些，愛刷徽章的人拿完就刪 app。兩邊都不對。

**唯一活下來的：貢獻徽章（建檔通過數）。**
因為提交新建檔要真的寫 metadata、要過官方審核。這個門檻是自然的，無法造假。

### 為什麼把照片設為私密？

**三個理由：**

1. **審美統一** — 官方圖色調一致、品質整齊；如果每人都放自己拍的爛照片，profile 看起來像雜貨箱。
2. **版權安全** — 海報圖有版權、你拍的照片也可能不小心侵權（背景入鏡、場景重製等）。不公開就不牽涉法律。
3. **反正驗證不了，乾脆別演戲** — 把照片定位為「個人紀念」而不是「所有權證明」，徹底解決徽章真假爭議。

### 為什麼別人看不到「你還沒收到的」？

**避免 FOMO 焦慮**。OpenSea 秀光所有 NFT，未擁有的人焦慮。Poster. 只秀你的成就，不秀你的缺口——別人看你 profile 看到的是「哇他好猛」，不是「他缺超多欸」。

### 那 Poster. 的「好玩感」從哪來？

**從工具本身的好用。**

- 瀏覽樹狀分類時看到罕見海報 → **知識性興奮**（Discogs、TMDb 都是這感覺）
- 翻牌時的動畫 + 自己的 collection 長大 → **UX 愉悅**（非競爭性）
- 發現官方沒建檔、自己補上且通過 → **貢獻成就感**（唯一真徽章）
- 看別人的 profile → 「哇他有這個」→ **社群啟發**（非嫉妒式炫耀）

這不是遊戲，是一個**有趣的工具**。跟 Google Maps、Notion、Letterboxd 同一類——使用者留下來是因為它**好用**，不是因為它**好玩**。

## 一頁備忘（給合夥人會議用）

- **產品定位**：海報收藏者的**工具** + 共建目錄社群。不是遊戲。
- **核心互動**：翻牌（信任制、免審核、免驗證、免徽章）
- **僅有的徽章**：貢獻建檔通過數（因為唯一有自然閘門、無法造假）
- **隱私**：我有什麼公開、我的自拍私密、我缺什麼隱藏、我的進度只有自己看得到
- **資料來源**：官方用 Google 試算表建目錄 → 後台補圖 → 使用者補漏
- **差異化**：不是 Instagram、不是 OpenSea、不是遊戲——是**實體海報收藏家的 Discogs**
- **為何不做遊戲化徽章**：物理海報無法免費驗證 → 任何「翻幾張、集滿、稀有度」的徽章都可以一鍵全破 → 發了等於送人免費勳章刷完就刪
- **為何工具就夠**：Discogs 800 萬使用者、Letterboxd 千萬使用者——都沒有遊戲化也都 scale
- **未來彩蛋**：線下活動 QR 簽到、官方印限定版附編號——這些是「有真實成本」才配得上有徽章
- **參考**：**Discogs**（主要學習對象）、Letterboxd（社交 feed 模型）、TMDb（共建目錄）
