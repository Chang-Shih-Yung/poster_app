# 吉卜力基準資料集 — Google Sheets 視覺化指南

更新 2026-04-24：CSV 結構大改版，**樹狀層級現在有自己的欄位**，配合
Google Sheets 幾個簡單設定就能一眼看出父子結構。

## 新版欄位設計（重點）

CSV 結構從舊版「`path` 字串一欄打包」→ 新版**每一層獨立欄位**：

| 欄位 | 層級 | 範例 | 意義 |
|---|---|---|---|
| A `studio` | 第 0 層 | 吉卜力 | 工作室 / 片商 / IP 持有者 |
| B `work_title_zh` | 第 1 層 | 神隱少女 | 作品中文名 |
| C `work_title_en` | 第 1 層 | Spirited Away | 作品英文名 |
| D `work_kind` | 第 1 層 | movie | 作品類別 |
| E `release_era` | 第 2 層 | 2001 日本首映 | 發行年代／場次 |
| F `variant_group` | 第 3 層 | 角色版 | 變體子類 |
| G `poster_name` | 第 4 層 | 千尋與白龍 | 葉節點：具體海報名稱 |
| H–S | 元資料 | 地區／年份／通路等 | 這張海報本身的細節 |

**肉眼看樹狀的祕訣在此**：
當 A 欄寫「吉卜力」、B 欄寫「神隱少女」、E 欄寫「2001 日本首映」、
F 欄寫「角色版」——**這四個欄位就畫出了整棵樹的縱深**，眼睛由左往
右掃就能讀。

## 匯入 Google Sheets 步驟

1. **新建試算表** → 命名為「Poster. 官方目錄」
2. **檔案 → 匯入 → 上傳** → 選 `ghibli_benchmark.csv`
3. 匯入類型選「**取代目前工作表**」
4. 語系選 **UTF-8**（中文才不會變亂碼）
5. 分頁改名為「**Ghibli 範例**」
6. 再新增一個分頁「**正式資料**」給編輯者擴充其他電影用

## 視覺化三連設定（強烈建議做）

### 🟢 1. 凍結標頭 + 第一欄

**檢視 → 凍結 → 1 列、 1 欄**。

做完之後：
- 往下滾欄位標題永遠在上面
- 往右滾 `studio` 欄永遠在左邊
- 再也不會迷失在 40 列欄位海

### 🟢 2. 條件式格式化 — 同一部電影同背景色

這一步是**畫出樹狀視覺**的關鍵：

**格式 → 條件式格式設定**

新增規則 1：
- 範圍：`A2:S200`（多選點，留空間給未來擴充）
- 公式：`=ISEVEN(COUNTA(UNIQUE(FILTER($B$2:$B2,$B$2:$B2<>""))))`
- 設背景色：**淺灰**（例如 `#F3F3F3`）

翻譯：這條公式會算「到這列為止出現過幾種 `work_title_zh`」，偶數的那
幾列（也就是第 2 / 4 / 6 部電影）上淺灰底。這樣 6 部電影**交替兩種
顏色**，一眼看到「這一塊都是同一部電影的變體」。

### 🟢 3. 群組收合（高階、但超爽）

**選取整部電影的列 → 右鍵 → 群組資料列 → 將這幾列分組**

對每部電影做一次。做完之後，每部電影左側會有一個 `-` 按鈕，點下去
**整部電影的所有變體會摺疊起來**，只剩該部的「標題列」。

結果：平常看起來像「6 部電影目錄」，點開單部才看到變體。這就是你
要的**樹狀收合感**，Google Sheets 原生支援，免外掛。

## 新增電影的流程

編輯者要補其他電影時，按這個順序寫：

1. 在「正式資料」分頁底端
2. **先插入一列空白**（視覺分隔，跟 Ghibli 範例一樣）
3. 新電影第一列：`studio` / `work_title_zh` / `work_title_en` / `work_kind` 四欄**必填**
4. 同一部電影接下來每列：這四欄**可以重複填**（最安全，也可以留空但排序/篩選會亂）
5. `release_era` 隨發行場次填（例如「2024 首映」「2025 4K 重映」）
6. `variant_group` 隨版本型態填（例如「正式版」「角色版」「teaser」「IMAX」）
7. `poster_name` 寫這張海報的**具體名稱**（最右一層的葉節點）

## 欄位對照（為什麼只有這些欄位，其他 DB 欄位在哪？）

編輯者在 Sheet **只管「海報本身長什麼樣」的欄位**。DB 裡還有更多，
但那些全部系統自動管：

| 不在 CSV 的欄位 | 為什麼不在 | 誰負責 |
|---|---|---|
| `id`, `work_id`, `parent_group_id` | 匯入時自動產生 UUID + 關聯 | 系統 |
| `created_at`, `approved_at`, `deleted_at` | 時間戳 | 系統 |
| `status`, `is_placeholder` | 審核 / 圖片狀態 | 後台流程 |
| `poster_url`, `thumbnail_url`, `blurhash` | 圖片相關 | 後台拖檔上傳時產生 |
| `favorite_count`, `view_count` | 使用者互動統計 | 系統累計 |
| `user_poster_state`, `user_poster_override` | 使用者端的擁有狀態 | 使用者互動 |

**編輯者腦中只有：「這張海報長什麼樣、是誰發的、什麼時候、什麼規
格、什麼通路」。其他都是系統的事。**

## ⚠️ 真實性說明

這份 CSV 的**電影 + 年份 + 大概變體類型**基於公開資料，但**個別
poster 的具體細節**（材質、特定通路、獨家名稱）**可能需要編輯者實
際查證後修正**。

標註為「**待編輯者確認**」的幾筆（例如「25 週年 IMAX 威秀獨家」
「2024 台灣 IMAX 獨家」），是典型會有的變體類型，但真正有沒有出過、
通路名是否正確，需要編輯者查官方發行紀錄。

**這是基準 / 學習資料，不是考古學級的正式版本紀錄。**

## 技術備註（給工程師）

- 編碼：UTF-8（Google Sheets 匯入要指定）
- 分隔符：`,`（逗號）
- 含逗號值：自動雙引號包起（CSV 標準）
- 空白列：純視覺用，匯入邏輯要跳過所有欄位全空的列
- 布林值：`TRUE` / `FALSE` 大寫
- 空值：留空即可（不要填 `NULL` 或 `-`）
- 多列同一部電影：`studio / work_title_zh / work_title_en / work_kind`
  四欄必須重複填（給匯入時做 upsert 用）

## 比舊版 CSV 好在哪（摘要）

| 舊版 | 新版 |
|---|---|
| `path = "A > B > C"` 一欄擠爆 | 每層一欄（release_era / variant_group / poster_name）|
| 肉眼分不出樹狀 | 眼睛由左往右掃就讀到層級 |
| 17 欄 | 19 欄（多 `studio` + 切開了 `path`）|
| 電影之間不易區分 | 空白列 + 條件底色，整齊可讀 |
| 不支援群組收合 | Google Sheets 群組收合原生支援 |
