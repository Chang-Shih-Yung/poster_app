# 吉卜力基準資料集（Ghibli Benchmark）

這份 CSV 是**給編輯者看的範例**——用吉卜力工作室的 6 部代表電影，
示範完整的樹狀分類、變體命名、欄位填寫邏輯。**不是正式發行資料**。

## 這份檔的用途

1. **教編輯者怎麼填**——看 37 列實際例子比看欄位規格有感
2. **給合夥人驗證 schema**——這些資料跑得過嗎？欄位夠用嗎？還缺什麼？
3. **當 benchmark 首批灌檔資料**——schema 定稿、後台建好後，這份是第一批匯入

## ⚠️ 資料真實性說明

這份 CSV 的**電影 + 年份 + 大概變體類型**是基於公開資料，但**個別
poster 的具體細節**（材質、特定通路、獨家名稱）**可能需要編輯者實
際查證後修正**。

標註為「**編輯者要確認**」的幾筆（例如「25 週年 IMAX 威秀獨家」「2024
台灣 IMAX 獨家」），是典型會有的變體類型，但真正有沒有出過、通路名
是否正確，需要編輯者查官方發行紀錄。

**這是基準 / 學習資料，不是考古學級的正式版本紀錄。**

## 欄位對照（為什麼是這 17 欄不是更多？）

DB 裡有很多欄位，但**編輯者不碰**——系統自動管：

| 不在 CSV 的欄位 | 為什麼不在 | 誰負責 |
|---|---|---|
| `id` | 匯入時自動產生 UUID | 系統 |
| `work_id`, `parent_group_id` | 系統看 `path` 自動建關聯 | 系統 |
| `created_at`, `approved_at`, `deleted_at` | 時間戳 | 系統 |
| `status`, `is_placeholder` | 審核/圖片狀態 | 後台流程 |
| `poster_url`, `thumbnail_url`, `blurhash` | 圖片相關 | 後台拖檔上傳時產生 |
| `favorite_count`, `view_count` | 統計 | 系統累計 |
| `uploader_id` | 歷史欄位 | v2 遺留 |

**編輯者腦中只有「這張海報長什麼樣、是誰發的、什麼時候、什麼格式」。
其他都是系統的事。**

## 如何用這份 CSV

### 1️⃣ 匯入 Google Sheets 當範例

```
Google Sheets → 檔案 → 匯入 → 上傳 → ghibli_benchmark.csv
→ 選「取代試算表」或「作為新工作表」
```

匯入後建議：
- 把這份設為獨立分頁「Ghibli 範例」（唯讀）
- 開一個空白分頁「正式資料」讓編輯者開始填其他電影

### 2️⃣ 給合夥人看欄位夠不夠

請他對照這 37 列想想：
- **有沒有哪個欄位用不到，可以砍？**
- **有沒有哪個資料類型欠一欄，要加？**
- **`path` 用 ` > ` 分隔夠不夠用，還是要改別的？**

這是最有效率的 schema 驗證方式——看真實資料在 schema 下長什麼樣。

### 3️⃣ 給編輯者當**欄位填寫教材**

他看 37 列範例就能學會：
- 同一部電影分幾個版本（1984 首映 vs 2005 台灣首映）
- 角色版 / 正式版 / teaser 的命名邏輯
- 獨家與否怎麼填（IMAX 威秀）
- 地區改 `region` 欄位（jp / tw / us / kr）

## 接下來可以擴充的方向

這份只是 6 部電影示範。編輯者後續可以：

1. **補齊吉卜力其他 18 部**（宮崎駿/高畑勳全作品）
2. **擴展到其他工作室**（新海誠、京阿尼、Pixar、迪士尼）
3. **深挖每部的稀有變體**（試映會、首映紀念品、限定印刷）

建議進度：**先求廣度（每部 1-2 個主要版本），再求深度（變體補齊）**。
廣度資料撐起 app 的「滿坑滿谷感」，深度資料給骨灰級收藏家挖。

## 不在這份範例裡、但可能未來要的欄位

這是我邊寫邊覺得「也許有用」的欄位，目前**刻意沒加**以維持 17 欄
簡潔性。交給合夥人定 schema 時決定：

| 欄位 | 用途 | 優先度 |
|---|---|---|
| `designer` | 海報設計師 / 畫師 | 中 |
| `print_run` | 印量（如「限量 500 張」）| 低 |
| `dimensions_mm` | 精確尺寸（比 b1/b2/b3 更準）| 低 |
| `release_date` | 具體發行日（不只年）| 中 |
| `official_price` | 原價（影廳贈品標 "贈"）| 低 |
| `aliases` | 其他名稱（日文原名 / 通稱）| 中 |

## 技術備註（給合夥人/工程師）

- CSV 編碼：UTF-8 (with BOM 或無，Google Sheets 都吃)
- 逗號值：含逗號的欄位自動用雙引號包起（CSV 標準）
- `path` 分隔符：` > `（space-greater-space），匯入邏輯照此切分
- 布林值：`TRUE` / `FALSE` 大寫（Google Sheets 轉 checkbox 友善）
- 空值：留空即可（不要填 `NULL` 或 `-`）
